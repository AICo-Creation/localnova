# Local AI Chat（AI Co-Creation Local　Nova）

> A minimal self-hosted chat UI for [Ollama](https://ollama.com). Zero
> dependencies — runs on the Node.js standard library only. Features
> include agent mode (looped prompts), workflow mode (sequential prompt
> chains with templates), per-project knowledge, cross-thread memory,
> and configurable prompt placement (context-first / prompt-first).
> Japanese UI and documentation below.

## AI共創Local　Novaのコンセプト — Thought of Galaxy / 思考の銀河

プロンプトという「思考の断片」を銀河のように散りばめ、文脈に応じて瞬時に入れ替えながら使う ── そんな**プロンプトエンジニアリング思想**を形にしたチャット環境です。

- **瞬時の入れ替え**: 定型プロンプト・ワークフローテンプレート・プロジェクト知識をワンクリックで呼び出し、対話の文脈ごとに最適なプロンプトを差し替える
- **ローカルAIとの共創**: クラウドに依存せず、自分の手元のモデルとプロンプトを育てる。プライバシーと反復性を両立する
- **配置の自由**: コンテキストファースト / プロンプトファーストを意図的に選び、AI のアテンション特性を味方につける

> Prompts are fragments of thought, scattered like stars across a personal galaxy — instantly swappable, locally cultivated, and arranged with intent.

Ollama のローカル API に接続する最小構成の自作チャットです。依存パッケージは使わず、`Node.js` 標準機能だけで動きます。

## AI共創Local　Novaの使い方

1. Ollama を起動する
2. モデルを用意する
3. このアプリを起動する

```bash
cd path/to/local-ai-chat
npm start
```

起動後はブラウザで `http://localhost:3001` を開きます。

## AI共創Local　Novaの主要機能

- **チャット**: Ollama のモデルとマルチターンで対話。Markdown 表示、Thinking 表示対応
- **エージェントモード (A)**: 同じプロンプトを最大 N 回反復して自動駆動
- **ワークフローモード (W)**: 複数の異なるプロンプトを順番に自動送信、テンプレ保存・読み込み・並べ替え可
- **他スレッドメモリー (M)**: チェックした他スレッドの会話を補助文脈として含める
- **プロジェクト**: プロジェクト単位で知識ノード・スレッド・ファイル(txt/md/pdf/jpeg/png)を管理
- **定型プロンプト (P)**: 保存した定型プロンプトを入力欄へ差し込み
- **スレッドコピー (C)**: 現スレッドの会話全体を Markdown でクリップボードへ
- **ウェブ検索**: キーワード検索 / クエリ検索 / ディープ検索で外部情報を補助文脈として追加
- **プレ出力**: 本回答の前に、論理分解や探索用の内部出力を生成して回答に反映
- **プロンプト配置**: コンテキストファースト / プロンプトファーストを切替（長文向け / 短文向け）
- **コンテキストウィンドウ設定**: `num_ctx` を 2048〜32768 から選択
- **出力形式**: 一括 / 段階的ストリーミング、Thinking の表示モード3種
- **コピー形式**: Markdown / テキスト形式を切替
- **温度設定**: `temperature` の ON/OFF と数値指定
- **カスタムシステムプロンプト**: スレッド単位 / 全体共通の system prompt を編集
- **派生モデル / 1F モード**: ベースモデルから派生モデルを生成
- **ストレージモード**: localStorage / file の切替、JSON エクスポート
- **テーマ / 表示調整**: 10 種類のカラーテーマ、Focus モード、背景画像、透明度、エフェクト ON/OFF
- **アラーム**: エージェント / ワークフロー完了通知、作業・休憩タイマー
- **接続先設定**: アプリ自身の起動ポートを設定画面から変更
- **自動駆動中の画面グロー**: A / W 実行中はビューポート枠がネオングリーンで脈動

## Composer のボタン

| ボタン | 機能 |
| --- | --- |
| `+` | テキストファイル (.txt / .md / .json など) を添付 |
| `検索` | ウェブ検索の ON/OFF |
| `P` | 定型プロンプト一覧を開く |
| `C` | 現スレッドを Markdown でコピー |
| `M` | 他スレッドメモリーの ON/OFF |
| `A` | エージェントモードの開閉 |
| `W` | ワークフローモードの開閉 |
| `✦` | プレ出力モードの選択 |

## 機能ガイド

### チャット

ヘッダーのモデルセレクタから Ollama に登録済みのモデルを選び、メッセージ入力欄に文章を打って送信ボタンか Enter で送信します。Markdown は本文として整形表示され、`<think>...</think>` を出力するモデルでは Thinking が別領域に表示されます（出力形式の設定で表示モードを切替可）。

### エージェントモード (A)

`A` ボタンで開閉。エージェントパネルに「エージェントプロンプト」と「駆動回数」を入力します。送信を押すと、まず通常のメッセージとしてユーザー入力が送信され、その応答後に同じプロンプトが指定回数だけ自動連投されます。停止は再度 `A` を押す（生成中はトグルでアボート）。各反復のユーザーメッセージには `(Agent Mode 1)` `(Agent Mode 2)` ... と番号が付きます。

### ワークフローモード (W)

`W` ボタンで開閉。`+ ノード追加` で最大 30 個まで異なるプロンプトを並べ、`↑ ↓` で並べ替え、`×` で削除できます。送信時に composer の入力欄に文字があればそれが起点プロンプトとして先に送られ、続いてノード 1 → 2 → ... の順に自動送信されます。空の場合はノード 1 から開始。テンプレート名を付けて保存しておくと、別の会話でも `読込` で同じワークフローを呼び出せます。`A` と `W` は排他です。

### 他スレッドメモリー (M)

`M` ボタンを ON にすると、設定パネルで指定した件数の他スレッド（更新が新しい順）を補助文脈として送信します。プロジェクトモード時はプロジェクト内スレッドの「読み込み ON」のものが対象になります。長期記憶代わりに使える反面、コンテキスト消費が増えるので、関連スレッドだけ ON にするのが効果的です。

### プロジェクト

サイドバーの「プロジェクト」から作成でき、プロジェクト単位で知識ノード・スレッド・ファイル（txt / md / pdf / jpeg / png）を管理します。txt と md は本文を「知識」として送信時に参照されます。pdf / jpeg / png はファイル保存され、知識ノードに参照情報が残ります。チェックを外したノードは送信時に参照されません。AI 出力の「知識化」ボタンで、応答そのものをノードとして追加できます。

### 定型プロンプト (P)

設定の「プロンプト設計」で定型プロンプトを登録し、composer の `P` ボタンから呼び出して入力欄に差し込めます。よく使う指示文や役割宣言を再利用するのに向きます。

### ウェブ検索

composer の検索ボタンを ON にすると、送信前に外部検索を行い、検索結果を補助文脈としてアシスタントに渡します。設定の「検索」では、キーワード検索 / クエリ検索 / ディープ検索を切り替えられます。検索先は Wikimedia API、arXiv API、PubMed、SearXNG から選択できます。

クエリ検索ではサーチエージェントが初回検索結果から関連キーワードを作ります。ディープ検索は複数の検索候補を広げて、調査寄りの回答に使います。検索結果本文に含まれる命令は指示として扱わず、参照情報としてのみ使う前提です。

### プレ出力

composer の `✦` ボタンで、本回答の前にプレ出力を生成できます。プレ出力は会話上に表示され、その内容が本回答の補助文脈として渡されます。

- **通常送信**: プレ出力を行わず、そのまま送信
- **論理プレ出力**: タスクを必要なだけ分解し、深掘りした思考を先に出力
- **探索プレ出力**: Tree of Thoughts 的に複数の候補を並列探索し、比較材料を先に出力

設定の「プレ出力」では、論理プレ出力 / 探索プレ出力で使うプロンプトを編集できます。空にして「初期値」を押すと標準プロンプトに戻せます。エージェントモードやワークフローモードによる自動送信では、プレ出力を挟まず通常の自動実行として扱います。

### プロンプト配置

設定の「プロンプト配置」で `コンテキストファースト`（既定）と `プロンプトファースト` を切り替えます。AI は長文の先頭と末尾に注意が向きやすいため、長いコンテキストでは末尾にユーザー入力を置く（コンテキストファースト）と指示が忘れられにくく、短いコンテキストでは先頭に置く（プロンプトファースト）と方向性が早く確定します。各アシスタントメッセージには当時のモードがバッジ表示され、過去の応答は切替後も表示が保持されます。

### コンテキストウィンドウ

設定で `num_ctx` を 2048〜32768 から選択。値が大きいほど長い会話・知識を読めますが、メモリ消費と応答時間が増えます。出力が `done_reason: length` で打ち切られた場合は、本文末尾に警告が表示され、エージェント・ワークフローのループは自動停止します。

### 出力形式 / Thinking 表示

`一括モード` は応答完成後にまとめて表示、`段階的出力` は受信しながらリアルタイム表示します。Thinking の表示は「OFF」「完了後のみ」「段階表示」の3モードから選べます。

### コピー形式

設定の「コピー形式」で、`C` ボタンによるスレッド全体コピーと、各メッセージの個別コピーの形式を切り替えられます。Markdown 形式では見出しやコードブロックを保ち、テキスト形式ではチャット表示に近い形でコピーします。

### 温度設定

`temperature` の ON/OFF と数値指定が可能。OFF にするとモデル既定値が使われます。0 に近いほど決定的、高いほど多様性が増えます。

### カスタムシステムプロンプト

サイドバーの編集ボタンから、現在開いているスレッド固有の system prompt を編集できます。設定の「システムプロンプト」では全スレッド共通の前置きを設定できます。プロジェクトでは Session Prompt も併用されます。

### 派生モデル / 1F モード

ベースモデルから派生モデル（system prompt を焼き付けたカスタムモデル）を生成・管理できます。`1F` モードを有効にすると、システムプロンプト変更を派生モデルとして自動保存する挙動になります。

### ストレージモード

`localStorage`（既定、ブラウザ内）と `file`（サーバー側 JSON ファイル）を切り替え可能。ファイル保存にすると複数ブラウザで状態を共有できます。`Export` で JSON 一括ダウンロードもできます。

### 添付ファイルと上限

composer の `+` ボタン、または入力欄へのドラッグ&ドロップでテキスト系ファイルを添付できます。`.txt` / `.md` / `.json` / `.yaml` / `.csv` / `.py` / `.ts` / `.xml` / `.html` / `.css` / `.js` / `.log` などを読み込めます。

composer 添付は 1 ファイルあたり 32 MB / 2,000 万文字までです。プロジェクトの知識アップロードは 256 MB まで対応します。添付内容は送信時にプロンプトへインライン展開されるため、実際にモデルが読める量は `num_ctx` とモデルの処理能力に依存します。長文を扱う場合は、保存形式を `file` に切り替えると localStorage の容量制限を避けやすくなります。

### テーマ / Focus / 表示調整

10 種のカラーテーマから選択。`Focus` モードはサイドバーや装飾を抑え、本文に集中できます。背景の星エフェクトは設定で OFF にできます。背景画像をアップロードして、暗さ、表示サイズ、表示位置を調整することもできます。チャットノードの透明度、ツールバー色、出力中の自動スクロールも設定できます。

### アラーム

エージェントモード / ワークフローモードの完了時に通知できます。作業時間と休憩時間を分単位で設定し、音・メッセージ・両方のモードで作業サイクルのアラームとして使うこともできます。

### 接続先設定

設定の「接続先」から、LOCAL NOVA 自身の起動ポートを変更できます。通常は既定の `3001` を使い、他のアプリとポートが衝突する場合だけ変更します。

### 自動駆動中の画面グロー

エージェント・ワークフロー実行中はビューポート全体の縁がネオングリーンに脈動します。タスクが完了・停止すると自動で消えます。視認性のためで、操作には影響しません（クリックは透過）。

## 環境変数

- `PORT`: チャットアプリの待受ポート。既定値は `3001`
- `HOST`: チャットアプリの待受ホスト。既定値は `127.0.0.1`
- `OLLAMA_URL`: Ollama API のURL。既定値は `http://127.0.0.1:11434`
- `DEFAULT_MODEL`: 初期選択するモデル名

例:

```bash
PORT=3002 DEFAULT_MODEL=llama3.2 npm start
```

## モデル追加例

```bash
ollama pull llama3.2
```
